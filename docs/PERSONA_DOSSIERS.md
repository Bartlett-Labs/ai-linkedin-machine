```

  │

  │   "This is exactly the kind of internal tooling that becomes a product.

  │
  ├─ [5-8 min] Jake Morrison (Skeptic) replies to Marcus:
  │   "Slow down. An internal script isn't a product. The gap between

  │    most side projects go to die."
  │
  ├─ [7-12 min] David Okafor (ROI Manager) comments:
  │   "The real question is throughput: how many hours per week does this
  │    reclaim? If the answer is >5, it pays for itself in a quarter."
  │
  ├─ [10-15 min] Dr. Priya Nair (Researcher) comments:
  │   "Interesting application of structured output parsing. Have you
  │    benchmarked hallucination rates on the numerical forecasting outputs?
  │    That's typically where LLM reliability breaks down."
  │
  ├─ [12-15 min] Alex Kim (Tinkerer) comments:
  │   "I built something similar for my personal finance tracking except
  │    I used a Raspberry Pi cron job hitting the Claude API at 3am lol.
  │    Way less polished but the core idea is the same."
  │
  └─ [optional] Rebecca Torres (Compliance) comments if topic touches
      data/privacy: "Worth noting that if those forecasts include PII or
      retailer-specific data, you'd want to ensure the LLM provider's
      data processing agreement covers your use case."
```

This creates a **conversation**, not just comments. The algorithm sees 5-6
distinct people debating within 15 minutes of posting = strong engagement signal.
